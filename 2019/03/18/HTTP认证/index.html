HTTP认证

发表于 2019-03-18 | 分类于 计算机网络

Basic认证

  • 浏览器弹出如下对话框，这就是Basic认证
  • 响应内容如下：
  • 用户将账号密码，用”:”分割，然后进行Base64编码，写在首部字段Authorization后，发给服务器
  • 缺点：Base64不是加密处理，等于明文，安全等级不够

DIGEST认证

  • 服务器发送临时质询码给客户端，nonce是随机字符串
  • 客户端首部字段包含账号密码等信息，response就是经过MD5运算过后的密码字符串
  • 缺点：安全性高于Basic认证，但不能防止客户伪装

SSL客户端认证

  • 服务器要求客户端提供客户端证书
  • 客户端发送证书信息
  • 证书验证通过后，领取客户的公开密钥，进行HTTPS加密通信
  • 另外：一般和表单认证组合使用，形成双因素认证。一方面验证账户和密码，一方面认证客户端计算机
